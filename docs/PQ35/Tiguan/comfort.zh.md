# 舒适

### 更改大灯清洗的触发频率 
标准频率（每 5 次）对某些人来说可能太频繁、太费水，因此可以更改洗挡风玻璃的中间触发次数。  
若设为「9」– 则大灯清洗每 10 次触发一次。  
``` yaml title="登录密码: 31347"
单元 09 → 适配: 
通道 26 → 输入所需值 → 保存 
```
完全关闭大灯清洗
``` yaml title="登录密码: 31347"
单元 09 → 编码 → 长编码:
字节 20 – 位 5: 停用 
```

### 下雨时自动关窗 
如果车辆降着车窗停放而下起雨，此功能会借助雨量传感器自动关窗，以防雨水淋湿内饰。 
``` yaml
单元 09 → 编码 → 长编码:
字节 4 – 位 5-6: 激活 
```
``` yaml
单元 09 → 编码 → 长编码 → 子块 rls:
字节 00 – 位 2: 激活 
```

### 后窗和后视镜加热的工作时长 
在北方地区，标准的 10 分钟加热可能不够，此时可延长加热时间（或缩短，若有人需要）。 
``` yaml title="登录密码: 31347"
单元 09 → 适配: 
通道 25 → 输入所需值 → 保存。所需值 
```
按公式计算：值 = 时间（秒）/ 40。即若要设定 20 分钟的加热时间，则 1200 / 40 = 30，输入值「30」。

``` yaml title="登录密码: 31347"
单元 09 → 编码 → 长编码:

```

### 通过门锁解除原厂防盗 
如果此功能激活，则可通过门锁解除车辆的原厂防盗，即此过程不涉及无线电子，仅靠钥匙金属片即可。也许对某些人有用，但还是最好关闭此功能，否则不法分子可通过普通机械破坏开门。 
``` yaml
单元 09 → 编码 → 长编码:
字节 6 – 位 2: 停用 
```

### 关闭雨刮的抽动 
在自动模式下工作时，雨刮在下位置总会往下多跳一下。 
``` yaml
单元 09 → 编码 → 子块 WWS → 长编码:
字节 1 – 位 2: 停用 
```

### 雨量传感器设定 
许多人抱怨雨量传感器工作不合逻辑，可以调节其灵敏度。 
``` yaml
单元 09 → 编码 → 子块 RLS → 长编码:
字节 2 – 位 0-7 → 选择所需值, 或手动指定。 
```
我通过实验确定了适合自己的最佳值 – CD。 

### 点火开启时用第二把遥控钥匙控制中控锁 
默认情况下，点火开启时无法用第二把遥控钥匙控制中控锁和舒适功能。  
此功能允许在点火开启或发动机运转时进行操作。 
``` yaml
单元 09 → 编码 → 长编码:
字节 4 – 位 2: 停用 
```

### 中控锁开/关的声音确认 
关锁时警笛响一声，开锁时响两声。  
开启： 
``` yaml
单元 09 → 编码 → 长编码:
字节 6 – 位 3: 激活 
```
关闭中控锁： 
``` yaml
单元 09 → 编码 → 长编码:
字节 5 – 位 1: 激活 
```
开启中控锁： 
``` yaml
单元 09 → 编码 → 长编码:
字节 5 – 位 4: 激活 
```

### 设防时折叠后视镜（自 ВАСЯ Диагност 15.6.2 / VCDS 15.7 版本起） 
原本那里是 90（关闭折叠） 
- 90 – 关闭折叠（默认）；  
- 95 – 设防时折叠，点火开启时展开；  
- 99 – 设防和撤防时折叠和展开。  
``` yaml
单元 42 → 封闭区域 → 输入「04354」→ 适配: 
(9)-System_Parameter_0-System Parameter Byte8 → 输入所需值 → 保存 
```
后视镜开关可以在任意位置。车门可以打开。无需启动车辆。 

### 驾驶员和乘客座椅加热记忆 
此功能可记忆前排座椅加热状态。在带自动启动的车上冬天非常有用。  
有三种选项：「not active」– 不激活，「active」– 激活，「active for 10 minutes」– 加热仅工作 10 分钟。 
``` yaml
单元 8 → 适配: 
通道「Retention of driver's seat heater level」→ 选择所需值 → 执行 
通道「Retention of passenger's seat heater level」→ 选择所需值 → 执行 
```

### 前排座椅自适应加热 
此功能在达到一定加热程度后自动降低前排座椅加热等级，即设为最大时，10 分钟后自动减小加热强度，以免「烤焦屁股」。 
``` yaml
单元 8 → 适配: 
通道「Seat heater performance reduction」→ 选择「active」→ 执行 
```
