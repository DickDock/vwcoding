# 舒适

### 第二代间接式胎压监测
第二代间接式胎压监测在 MaxiDot 上显示具体车轮的压力变化。  
基于每个车轮的转速进行判定，前提是变化不小于 0,5 атм。  

!!! note "说明"
    无需安装额外设备。  
    仅可在带 MaxiDot、仪表盘型号不低于 843 Q、带 ESC 且 3 单元中有第 18 或 19 字节的车辆上激活。
  
``` yaml
单元 19 → 设备列表:
4C – 第二代胎压监测: 激活
---
单元 17 → 编码 → 长编码:
字节 2 – 位 0: 激活
---
单元 3 → 编码 → 长编码:
字节 17 – 位 2: 激活
```
对于带 19 字节的 ABS <=BL 单元（首位「0」末位「18」）
``` yaml
单元 3 → 编码 → 长编码:
字节 16 – 位 1: 激活
单元 3 → 编码 → 长编码:
字节 16 – 位 2: 激活
```
对于带 20 字节的 ABS =>BM 单元（首位「0」末位「19」）
``` yaml
单元 3 → 编码 → 长编码:
字节 19 – 位 0: 激活
单元 3 → 编码 → 长编码:
字节 19 – 位 1: 激活
```

!!! warning "注意"
    激活后需进入 3 单元并清除故障列表。

!!! note "说明"
    仅适用于 Škoda Octavia A5FL 1.4 TSI。

``` yaml
单元 17 → 适配:
通道「IDE03721」→ 设为值「active」→ 执行
```

### MaxiDot 上的自动后雨刮
在 MaxiDot 菜单「舒适与照明」中会出现开启/关闭后雨刮自动模式的选项，该模式表现为：当前雨刮以连续模式工作超过 1 分钟时，后雨刮每分钟工作 1 次。
``` yaml
单元 09 → 编码 → 长编码:
字节 25 – 位 0: 激活
```

### 关闭前雨刮的可变停靠位置

``` yaml
单元 09 → 编码 → 1 单元 Wischer → 长编码:
字节 1 – 位 2: 停用
```

### 关闭前雨刮的停车区位置
长时间停车时（熄火后），雨刮下降到一个特殊位置，略低于工作区。
``` yaml
单元 09 → 编码 → 1 单元 Wischer → 长编码:
字节 2 – 位 5: 停用
```

### 前雨刮的附加回刮
此功能可在使用玻璃水后几秒自动擦掉残滴。

``` yaml

字节 1 – 位 3: 激活
```

### 后雨刮的附加回刮
往后窗喷水后，标准的一次刮拭常常不够，往往就在 10 秒后，水滴又会狡猾地流到已干的玻璃上。  
此功能可在使用玻璃水后几秒自动擦掉残滴。
``` yaml
单元 09 → 编码 → 长编码:
字节 21 – 位 4: 激活
```

### 更改大灯清洗的触发频率
标准频率（每 5 次）对某些人来说可能太频繁、太费水，因此可以更改洗挡风玻璃的中间触发次数。  
若设为「3」，则大灯清洗每 4 次触发一次；若设为「7」，则大灯清洗每 8 次触发一次。
``` yaml
单元 09 → 适配: 
通道 26 → 测试 → 输入所需值 → 保存
```


### 在显示屏上显示油箱可加注余量
油箱剩余空间 – 可加注升数的近似值。刻度步长 – 5 升。  
油箱加满时显示「---」。  
读数非常近似，相对实际状态可能有 +/- 10 升的偏差。  

!!! note "说明"
    仅适用于 2012 款车辆。
  
``` yaml
单元 17 → 适配: 
通道「IDE04848」→ 设为值「display」/「ok」→ 执行
```

``` yaml
单元 09 → 编码 → 长编码:

```

### 后视镜电动折叠
如果有人需要关闭后视镜电动折叠功能，则应从当前值中减去「4096」；相应地，如需开启此功能，则把「4096」加到当前值上。

!!! note "说明"
    用此方法关闭后视镜电动折叠是彻底关闭，即无论防盗、遥控器还是后视镜控制摇杆都无法折叠。

``` yaml
单元 42 → 编码 → 字段 1 → 设为所需值 → 执行
单元 52 → 编码 → 字段 1 → 设为所需值 → 执行
```

### 倒车时右侧后视镜下翻
挂倒挡时，右侧后视镜下翻到下位置，以便更好地观察右后轮附近的泊车区域。  
熄火或以超过 20 km/h 向前行驶时，后视镜回到原位。

!!! note "说明"
    仅适用于带电动调节、且 52 单元中有长编码的后视镜。
  
``` yaml
单元 09 → 编码:
字节 4 – 位 3: 激活
单元 52 → 编码:
字节 4 – 位 2: 激活
```

### 设定后视镜和后窗加热时长
在北方地区，标准的 10 分钟加热可能不够，此时可延长加热时间（或缩短，若有人需要）。

!!! note "说明"
    所需值按公式计算：值 = 时间（秒）/ 40。  
    即若要设定 20 分钟的加热时间，则 1200 / 40 = 30，输入值「30」。  
    只能输入整数值！
  
``` yaml
单元 09 → 适配: 
通道 25 → 测试 → 输入所需值 → 保存
```

!!! warning "注意"
    默认值为「15」。如无必要，不要滥用加热时间，因为玻璃和保险丝可能承受不了长时间的加热工作。

### 前排座椅加热记忆
此功能可记忆前排座椅加热状态。在带自动启动的车上冬天非常有用。

!!! note "说明"
    乘客座椅加热记忆仅在 2013 款车辆上可开启（在 3T0 907 044 BS 单元上确认有效），软件版本不低于 12.12 时有保证。 
    不同单元的软件版本可能对乘客座椅的激活有限制。  
    有三种选项：「not active」— 不激活，「active」— 激活，「active for 10 minutes」— 加热仅工作 10 分钟。
  
``` yaml
单元 8 → 适配: 
通道「IDE01067」→ 选择所需值 → 执行
通道「IDE07948」→ 选择所需值 → 执行
```

!!! warning "注意"
    第一个编码激活驾驶员座椅记忆，第二个激活乘客座椅记忆。

### 校正车速表读数
按安全规范，车速表读数相对实际车速略偏高。  
不同车辆设有不同的读数偏高等级。

!!! note "说明"
    来自 Škoda Octavia (01) 的脉冲数在 100 km/h 时读数偏高 ≈ 8 km/h，来自 Škoda Fabia (02) - ≈ 15 km/h。  
    采用 VW Golf (07) 的脉冲数时读数最准确，但此值不一定能写入所有单元。

``` yaml
单元 17 → 编码 → 长编码:
字节 0 – 位 0-3 → 改为所需值
```

### 空调适配
此适配依次开关所有风门、活动空调的所有内部机构，检查所有按钮和指示灯状态，并进行其他操作，可活动冻结或卡滞的空调。  
整个过程不到 1 分钟。
``` yaml
单元 8 → 04 – 基本设置 → 
通道「IDE01546」→ 读取 → 等待状态「成功完成」
```
